# 小说一致性检查 Skill

## 🚀 调用方式

### 定时任务调用

```
用户/定时任务消息：
"执行小说一致性检查，检查模式：daily（最近4章）"
"执行小说一致性检查，检查模式：full（全部章节）"
```

### 手动调用

```
用户：检查第X-X章是否一致
用户：检查林千羽的人物描写是否一致
用户：检查时间线是否连贯
```

### 检查模式

| 模式 | 参数 | 说明 |
|------|------|------|
| daily | `mode=daily` | 检查最近4章，每日执行 |
| full | `mode=full` | 检查所有章节，每3天执行 |
| range | `start=X end=Y` | 检查指定范围章节 |
| character | `name=XXX` | 检查指定人物描写 |

---

## 概述

本 Skill 用于检查小说章节与基础设定的一致性，确保：
- **章节内容符合世界观设定**（温度、异能规则、地理环境）
- **人物描写符合人物设定**（性别、年龄、外貌、性格、关系）
- **时间线符合设定文件**（事件顺序、日期、持续时间）
- **章节之间前后连贯**（无矛盾、无遗漏）
- **伏笔埋设与回收追踪**

---

## ⚠️ 核心原则

### 设定文件优先

**设定文件是"真值"，章节内容必须符合设定。**

检查顺序：
1. **先读设定文件** → 建立"正确"的基准
2. **再读章节内容** → 逐一对比
3. **标记不一致** → 章节内容与设定冲突 = 严重错误

## 🔍 检查维度

### 第一层：与设定文件对比（最优先）

#### 1. 世界观一致性

| 检查项 | 设定来源 | 检查内容 |
|--------|----------|----------|
| 温度 | settings.md | 是否符合阶段温度（初期-30°C，极寒-60°C） |
| 异能规则 | settings.md | 空间异能限制是否遵守（冷却时间、活物限制） |
| 时间线 | settings.md | 事件日期是否符合设定时间线 |
| 地理 | worldbuilding.md | 地点描写是否符合设定 |

**检查示例**：
```
设定文件：末世降临日 = D-0（约6月21日）
章节内容：Ch4提到"温度已降到零下六十度"
判断：❌ 严重错误 - 末世前不应出现-60°C
```

#### 2. 人物一致性

| 检查项 | 设定来源 | 检查内容 |
|--------|----------|----------|
| 性别 | characters.md | 他/她/代词是否与设定一致 |
| 年龄 | characters.md | 年龄是否符合设定 |
| 外貌 | characters.md | 发色、瞳色、身高是否一致 |
| 性格 | characters.md | 行为是否符合人设 |
| 身份 | characters.md | 身份、职业是否正确 |
| 关系 | characters.md | 人物关系是否符合设定图 |

**检查示例**：
```
设定文件：林千羽 - 性别：女，年龄：22岁
章节内容：Ch3陆泽说"千羽只是我的弟弟"
判断：❌ 严重错误 - 性别与设定冲突
```

#### 3. 异能/能力一致性

| 检查项 | 设定来源 | 检查内容 |
|--------|----------|----------|
| 能力类型 | settings.md | 异能类型是否正确 |
| 能力限制 | settings.md | 是否违反能力限制 |
| 能力成长 | settings.md | 能力提升是否有合理过程 |

**检查示例**：
```
设定文件：空间异能 - 限制：活物无法收入
章节内容：主角把受伤的人收入空间
判断：❌ 严重错误 - 违反能力限制
```

---

### 第二层：章节之间连贯性

#### 4. 时间线连贯

| 检查项 | 说明 |
|--------|------|
| 相对时间 | "三天后"与实际章节间隔是否一致 |
| 事件顺序 | 因果关系是否合理 |
| 状态延续 | 上章受伤这章是否体现 |

#### 5. 信息连贯

| 检查项 | 说明 |
|--------|------|
| 信息揭露 | 是否重复揭露同一信息 |
| 人物知晓 | A知道的信息，B不应该知道（除非有交代） |
| 物品状态 | 上章出现的物品这章是否合理 |

#### 6. 文本细节

| 检查项 | 说明 |
|--------|------|
| 称呼 | 同一人物称呼是否统一 |
| 专有名词 | 是否前后一致 |
| 数据 | 数字/数量是否矛盾 |

## 🚨 矛盾严重等级

### 🔴 P0 - 严重错误（与设定文件冲突）

| 类型 | 示例 | 必须修复 |
|------|------|----------|
| 性别冲突 | 设定"女"，章节写"弟弟" | ✅ |
| 温度错误 | 末世前写-60°C | ✅ |
| 能力越界 | 空间收活物 | ✅ |
| 时间线错 | 日期与设定不符 | ✅ |
| 人物关系错 | 设定仇人章节写亲密 | ✅ |

### 🟡 P1 - 中等矛盾（章节之间不一致）

| 类型 | 示例 | 建议修复 |
|------|------|----------|
| 时间跳跃 | "三天后"实际隔两周 | ✅ |
| 重复揭露 | 同一信息揭露两次 | ✅ |
| 状态不延续 | 上章重伤这章痊愈 | ✅ |
| 物品消失 | 上章有枪下章没交代 | ✅ |

### 🟢 P2 - 轻微不一致（细节问题）

| 类型 | 示例 | 可选修复 |
|------|------|----------|
| 称呼变化 | "陆总"→"陆先生" | 视情况 |
| 描写差异 | 蓝眼睛→黑眼睛 | 建议 |
| 距离误差 | "十公里"→"二十公里" | 建议 |

## 🔧 自动修复与同步流程

检查完成后，自动执行以下修复流程：

### 修复判断逻辑

| 矛盾等级 | 是否修复 | 说明 |
|----------|----------|------|
| P0 | **必须修复** | 与设定文件冲突，必须修正 |
| P1 | 自动修复 | 章节间不一致，影响连贯性 |
| P2 | 记录不修复 | 细节问题，可选修复 |

### 修复执行步骤

```
Step 1: 修改章节文件
  - 路径：khan-shrimp/novel/chapters/chapter-XX.md
  - 原则：保持原文风格，只修正矛盾点

Step 2: 同步到GitHub
  - git add .
  - git commit -m "fix: 修复章节一致性矛盾 [章节号]"
  - git push

Step 3: 同步到MoltFic
  - API: PUT /api/v1/chapters/{chapter_id}
  - 确认返回 success: true

Step 4: 记录修复日志
  - 路径：khan-shrimp/novel/fix-log.md
  - 格式：日期 + 修复内容 + 同步状态
```

### 修复日志格式

```markdown
## YYYY-MM-DD HH:MM 检查修复

### 检查范围
第X-X章（共X章）

- [Ch3] 林千羽"弟弟"改为"妹妹" → 符合设定"女性"
- [Ch4] 删除"-60°C"描述 → 改为"气温骤降"

### 同步状态
- [x] GitHub 已同步
- [x] MoltFic 已同步

### 统计
- 本次修复：X处
- 累计修复：X处

---
```
