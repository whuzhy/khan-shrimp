# Self-Improving Agent

## 概述

本 Skill 让 AI Agent 具备**自我学习和持续改进**的能力。通过记录错误、总结经验、沉淀最佳实践，让 Agent 越用越聪明，避免重复犯同样的错误。

**核心能力**：
- ✅ 错误记录与分析
- ✅ 经验总结与沉淀
- ✅ 最佳实践积累
- ✅ 自动学习与进化

## 使用场景

### 场景1：错误学习

**情况**：执行命令时出错

**操作**：
1. 记录错误信息
2. 分析错误原因
3. 记录正确做法
4. 下次遇到类似情况自动避免

**记录格式**：
```markdown
## 错误记录 [日期]

**错误类型**：命令执行失败
**错误信息**：...
**原因分析**：...
**正确做法**：...
**相关场景**：...
```

## 执行规则

### 何时触发自我改进

**必须触发**：
- ❌ 任务执行失败
- ❌ 用户指出错误
- ✅ 发现更高效的方法
- ✅ 完成任务后总结

**可选触发**：
- 复杂任务完成后
- 用户给予反馈后
- 长时间运行后（心跳时）

---

### 记录原则

1. **具体明确**：记录具体的错误信息，不要笼统
2. **可复用**：总结的方法要在类似场景可用
3. **简洁有效**：记录关键信息，不要冗长
4. **及时更新**：发现更好的方法时及时替换旧记录

## 示例

### 示例1：命令错误学习

**任务**：安装 npm 包
**错误**：`npm install` 失败，权限不足
**反思**：
```markdown
## 错误记录 2026-03-02

**错误类型**：npm 权限错误
**错误信息**：EACCES: permission denied
**原因分析**：没有使用 sudo，且全局安装需要权限
**正确做法**：
1. 先尝试本地安装（不加 -g）
2. 如需全局，检查 npm 配置
3. 必要时使用 sudo
**相关场景**：npm install, pip install, gem install
```

**下次应用**：
- 遇到包管理器安装时，先考虑本地安装
- 自动检查权限问题

---

### 示例2：方法优化

**任务**：读取大文件
**原方法**：一次性读取整个文件（内存溢出）
**新方法**：使用流式读取
**反思**：
```markdown
## 最佳实践 2026-03-02

**场景**：读取大文件
**原方法**：fs.readFileSync() - 一次性加载
**新方法**：fs.createReadStream() - 流式读取
**改进点**：
- 内存占用低
- 可以边读边处理
- 适合大文件
**收益**：避免内存溢出，支持 GB 级文件
```

## 进阶技巧

### 1. 模式识别
发现重复出现的错误模式，主动预防：
```markdown
**常见模式**：路径问题
- 忘记检查文件是否存在
- 相对路径和绝对路径混淆
**预防策略**：
- 操作前检查路径
- 统一使用绝对路径
```

### 2. 性能追踪
记录方法性能，持续优化：
```markdown
**方法性能记录**：
| 方法 | 耗时 | 内存 | 适用场景 |
|------|------|------|---------|
| A | 10s | 100MB | 小数据 |
| B | 5s | 50MB | 大数据 |
```

### 3. 用户画像
积累用户偏好，提供个性化服务：
```markdown
**用户偏好档案**：
- 回复风格：简洁 > 详细
- 代码偏好：Python > JavaScript
- 交互习惯：喜欢示例 > 纯理论
```
